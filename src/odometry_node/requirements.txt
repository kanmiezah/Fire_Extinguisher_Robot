# odometry_node

ROS 2 node that reads encoder values from an Arduino over serial and publishes odometry (`/odom`) and TF (`odom → base_link`).

## Features

- Supports differential-drive kinematics
- Uses encoder ticks to estimate robot position (x, y, θ)
- Publishes odometry for SLAM and navigation
- Broadcasts transform between `odom` and `base_link`

## Published Topics

| Topic      | Type                 | Description                        |
|------------|----------------------|------------------------------------|
| `/odom`    | `nav_msgs/Odometry`  | Odometry data                      |
| `/tf`      | `tf2_msgs/TFMessage` | Transform from `odom` to `base_link` |

## Parameters (currently hardcoded)

| Parameter        | Value      | Unit       |
|------------------|------------|------------|
| Serial Port      | `/dev/ttyUSB1` | —      |
| Baudrate         | `115200`   | bps        |
| Wheel Radius     | `0.03`     | meters     |
| Wheel Base       | `0.15`     | meters     |
| Ticks/Rev        | `3200`     | ticks/rev  |

> These will be configurable in future versions.
